## Purpose

This document defines the **pre-testing adaptation tranche** after Gate 58.

The purpose of this tranche is to make the deterministic desk-cognition repo honest about **how it is allowed to adapt** before any historical replay optimisation, candidate discovery, or live-paper falsification begins. The tranche therefore freezes:

- stable doctrine and bounded variable posture
- coefficient and modifier ontology
- non-action, conflict, and discretion law
- stability-metric and corridor law
- review-eligibility and candidate-governance law
- desk-calendar, event, and precursor context required by those laws
- bounded runtime integration surfaces for state-conditioned modifiers
- walk-forward review-horizon discovery machinery for later testing

This is **not** a repo rewrite and **not** a licence to start fitting coefficients. It is a controlled extension of the current deterministic desk-cognition architecture so that the repo can preserve richer context and vary operating posture lawfully without mutating cognition grammar or drifting into informal retuning.

The governing workflow remains:

`calendar/horizon -> temporal_context -> market_regime_context -> options_flow_context -> posture_risk_permission -> playbook_eligibility -> expression_execution -> review_explanation`

## Tranche doctrine

1. The repo is not discovering a new trading philosophy per session. It preserves a stable cognition grammar and allows only bounded, deterministic variation in **operating posture** under approved state.
2. Historical replay and live unknown data are the same shape of evidence to the model; the distinction is governance. Historical replay is the research/discovery surface. Live paper is the falsification/promotion surface.
3. Runtime never invents new coefficients or hidden policy in place. Runtime may only apply approved invariant, baseline, and state-conditioned policy surfaces frozen by earlier gates.
4. Review does not imply change. “No change” is a valid governed review outcome.
5. No coefficient search, candidate adjudication, or horizon-discovery testing begins until this full tranche is closed on `main`.
6. One branch per gate. Finish the gate, pass targeted validation, merge to `main`, and only then open the next gate branch.
7. Even doctrine/document gates may require consistency tests if they alter binding language, planning order, or canonical vocabulary.
8. Any gate that reveals ontology failure, missing-module truth, or invalid assumptions must emit a research-reset note rather than smuggling redesign into the current gate.
9. No gate may reorder cognition grammar or introduce hidden runtime routing outside the typed workflow without explicit normative changes.
10. The purpose of this tranche is to formalise lawful adaptation before testing, not to smuggle testing conclusions into planning docs.

## Non-goals for this tranche

- starting historical coefficient search before adaptation doctrine is frozen
- allowing live paper to become in-place coefficient discovery
- replacing typed packet surfaces with loose dict payloads
- allowing state-conditioned modifiers to alter cognition grammar order
- inventing an unbounded event/news ontology in one pass
- creating a huge free-form precursor universe without bounded desk rationale
- collapsing review into narrative commentary instead of evidence objects
- implementing a global aggressiveness dial before the lower-level posture system exists

## Parked future direction (not part of this tranche)

A future global posture-control manifold / aggressiveness dial may be explored later, but **only after** baseline coefficients, state-conditioned modifiers, review corridors, candidate governance, and review-horizon discovery are all proven. It is intentionally **not** a gate in this tranche.
