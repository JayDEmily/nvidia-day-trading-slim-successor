# 2026-04-02 Coefficient Architecture Consolidation Gates v1

## Purpose

Create the active planning pack that turns the coefficient architecture crosswalk into executable repo work without skipping the repo's gate / leaf / audit discipline.

This pack exists to do seven things only:
1. preserve the useful doctrine, vocabulary, seam-hardening work, and workbook evidence already present in the repo rather than inventing a new architecture from scratch;
2. consolidate the coefficient architecture for Workstreams 1-4 using repo-native vocabulary and explicit boundary law;
3. collapse the two-coefficient-world confusion into one governed live runtime authority plus explicit reference / provenance classes;
4. make owner-stage truth executable by reconciling owner labels, actual consumers, and activation state for every admitted mutable surface;
5. separate opportunity shaping from caution shaping without multiplying live knobs or pretending the independent risk lane already exists;
6. route the later coding tranche in an order that preserves workbook gold, raw-versus-derived discipline, and stage-purity law;
7. close with an audit gate that checks the pack's own coverage, vocabulary discipline, routing truth, and anti-drift integrity before any later coding thread claims the plan is complete.

## Why this pack exists

The repo is not missing coefficient architecture entirely. It already contains the core ontology in `docs/01_NORMATIVE.md`, the stage grammar in the runtime spine, a narrowed governed runtime authority in `config/coefficient_authority.v1.yaml`, a broader salvage/reference registry in `config/coefficients_registry.example.yaml`, and a surprisingly rich workbook that behaves like a practical migration ledger.

The defect is convergence, not absence.

Today the repo still makes future-you work too hard to answer four simple questions:
- what is live runtime coefficient truth versus workbook / salvage reference;
- which stage really owns each mutable surface;
- whether a governed surface is dynamically active or just admitted at baseline;
- and where opportunity shaping stops and caution shaping begins.

The workbook strengthens the case for a new pack rather than weakening it. It shows that the repo already has raw-versus-derived doctrine, stage-purity doctrine, a candidate coefficient universe, and a richer upstream opportunity path through better inputs and playbook routing. That gold needs to be promoted into repo-native planning law instead of staying half-visible inside a spreadsheet.

## Scope

In scope:
- planning-only execution for Workstreams 1-4;
- repo-native consolidation of the coefficient architecture using existing vocabulary first;
- explicit live/reference/provenance/deferred classification for coefficient surfaces;
- owner-stage and activation-state planning for all admitted mutable runtime surfaces;

- broad code changes in this bootstrap gate;
- widening the live mutable surface set during bootstrap;
- treating workbook sheets as runtime authority;
- silently amending normative doctrine when the actual need is better planning law;
- claiming serial conservatism is solved here merely because later gates mention it.
