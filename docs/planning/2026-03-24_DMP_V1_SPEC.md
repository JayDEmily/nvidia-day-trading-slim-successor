# 2026-03-24 DMP v1 Specification

Status: Active Gate 8 implementation note; canonical live runtime packet surface for the workflow-modification tranche  
Scope: internal desk-runtime packet wrapper only

## 0. Gate 54 binding decision

For the workflow-modification tranche closed through Gate 54, DMP v1 remains the **canonical live packet producer surface**.

DMP v2 packets may exist as derived migration/inspection companions, but they are not the primary binding producer contract until a dedicated DMP-v2 promotion gate proves and promotes that change.

## 1. Purpose

Desk Module Protocol (DMP) v1 is the internal typed envelope for desk-runtime stage outputs and checked-in module artefacts.

It exists to add deterministic metadata around an already-typed payload. It does **not** replace the payload model with an untyped dictionary.

## 2. Required envelope fields

A DMP v1 packet carries these top-level surfaces:

- `protocol_version`
- `packet_identity`
  - `packet_id`
  - `emitted_at`
- `grammar_role`
- `behaviour_class`
- `schema_identifiers`
  - `payload_model_name`
  - `payload_module_path`
  - `input_model_name` (optional)
  - `output_model_name` (optional)
- `stack_id`
- `coefficient_set_id`
- `dependencies`
- `trace_references`
- `trader_summary`
- `payload`

`stack_id` and `coefficient_set_id` remain first-class top-level fields because later replay and comparison work must be able to cite them without scraping nested payload state.

## 3. Payload rule

The payload remains a concrete repo-native Pydantic model.

For Gate 8, that means DMP wraps the current cognition outputs and calibration/replay artefacts without renaming or replacing those models.

## 4. Non-goals

DMP v1 does **not**:

- define an external transport protocol;
- replace MCP;
- define frontend payload contracts;
- define live broker payload contracts;
- force a rewrite of the existing cognition output models;
- solve every trace and replay lineage problem in Gate 8.

MCP remains external. DMP remains internal.

## 5. Gate-8 stop rule

If the packet design starts widening into transport orchestration, live integration, or wholesale payload rewrites, Gate 8 has drifted and the work should stop.
